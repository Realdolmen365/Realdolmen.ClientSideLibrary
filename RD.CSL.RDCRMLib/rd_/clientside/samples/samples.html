RD.CSL

This is the main JScript library to include on your form and/or html webresources

Following components are available:

Name Type Parent NameSpace Description
RD Namespace / The root namespace of this library.
Log Function RD Logs a message to the console.
GetContext Function RD Gets the XRM context object if it is available
SetExecutionContext Function RD Set the XRM execution context if needed via checkbox (9.0 only)
GetClientURL Function RD Gets the current client URL.
LoadScript Function RD Loads a script into the window.
Helpers Library Namespace RD.Helpers Provides a set of common helper functionalities.
Form Type Namespace RD Provides a set of properties to check the current form type.
Custom Controls Namespace RD Add functionality to the CRM Custom Controls to use in a webresource.
CrmFetchKit Namespace RD A third party library that provides a set of functions to Query CRM using FetchXml.
REST Namespace RD A third party library that provides a set of functions to comunicate with CRM using oData.

Name Description

RD.Libraries.Helpers.DependantPicklists()

Parameters

ExecutionContext CRM 365 v9.0

Type: String

The execution context as first parameter (check). Not required for CRM versions < 9.0

Parent Field Name

Type: String

The logical name of the field that filters the other field.

Dependant Field Name

Type: String

The logical name of the field that will be filtered.

Mapping

Type: Array

The mapping between the values of the parent field and the values of the filtered field.

Example

JavaScript

var mapping = {
    200000003: [200000009, 200000010],              //German values
    200000000: [200000000, 200000001, 200000002],   //English values
    200000002: [200000006, 200000007, 200000008],   //French values
    200000001: [200000003, 200000004, 200000005]    //Dutch values
};
RD.Helpers.DependantPicklists("new_parentfield", "new_filteredfield", mapping);
or
RD.Helpers.DependantPicklists(executionContext, "new_parentfield", "new_filteredfield", mapping);

Custom Controls

The Custom Controls namespace provides a function to initialize a custom form that has to behave like a CRM form.

RD.CustomControls.Initialize()

Call this function after the page loads, it will initialize all functionality to mimic CRM fields.

Example

JavaScript

//Wait for the DOM to load
$(document).ready(function () {
    //initialize the actions for the custom controls
    RD.CustomControls.Initialize();
});

Requirements

To mimic the CRM fields you have to include jQuery & the CRM ClientGlobalContext.js.aspx page before the RD.CSL.js library.

jQuery UI is optional and only necessary for the date field.

HTML

<script src="../../../ClientGlobalContext.js.aspx" type="text/javascript"></script>
<script src="../scripts/jQuery.js" type="text/javascript"></script>
<script src="../scripts/jquery_ui_1.10.4.custom.js" type="text/javascript"></script>
<script src="../RD.CSL.js" type="text/javascript"></script>

Include the CRM Style css to get mimic the CRM Layout.

CRM 2015 Themes

Theme.css is an optional file that can be included if you run CRM 7.1 (Carina).

When you publish a theme this file will be updated so that the custom controls will still resemble the CRM fields.

Theming capabilities for custom controls can be installed using the RDCustomControlTheming solution.

Extra Body classes

If you want to use the CRM 2013 style give the body of the page the class "crm2013".

If you want to use the page embedded in a form give the body of the page the class "onform". This removes the left padding.

HTML

                
<link href="../css/CRM_Style.css" rel="stylesheet" />
<link href="../css/Theme.css" rel="stylesheet" />

Control Types & Features

Following control types and features are available:

Name Description
Single line of text A regular text input (url, email & telephone formats are available)
Lookup field Holds a reference to another entity record.
Option Set Provides a set of options to select from.
Date field Enables you to select a date.
Checkbox Simple true/false input.
Required Fields A required field is marked with a red * next to the label.
Locked Fields A locked field is marked with a lock icon next to the label and is read only.
Complete Sample A complete page example.

Note: It is best to include the controls in a table, this will make sure the controls are arranged correctly.

CrmFetchKit

The RD.CRMFetchKit namespace contains the CrmFetchKit third party library.

It holds several methods to retrieve data as wel as the Assign message.

Name Description
GetById Instead of create a fetchxml query for a very simple query, this method should be used to load an records based on the id.
GetByIdSync Same method as GetById except with synchronous execution.
Fetch With the Fetch method is it possible to execute fetch-xml based query.
FetchSync Same method as Fetch except with synchronous execution.
FetchMore In a situation where the developer needs more control over the loaded data, the FetchMore method should be used.
FetchMoreSync Same method as FetchMore except with synchronous execution.
FetchAll To address the 5.000 records query limit of Dynamics CRM (a single request return at a maximum 5.000 records) provides the CrmFetchKit with the FetchAll method an option to load all records retunred by an query.
FetchByPage This method allows the load of records per page-number.
Assign The Assign method allows the modification of the owner of an CRM record.
AssignSync Same method as Assign except with synchronous execution.
 

RD.CRMFetchKit.getById()

Parameters

Entityname

Type: String

The logical name of the entity you want to retrieve.

Id

Type: String

The string representation of the Guid of the record you want to retrieve.

Columns

Type: Array (String)

An Array containing the columns you want to retrieve.

Example

JavaScript

var accountid = '06887701-2642-4a53-99ba-c24ce1a5688b', 
    columns = ['name', 'donotfax', 'donotemail', 'createdon'];
RD.CRMFetchKit.GetById('account', accountid, columns).then(function(account) {
    console.log(account.getValue('name'));
});

RD.CRMFetchKit.FetchMore()

Parameters

FetchXML

Type: String

The fetch XML determining what you want to retrieve.

Returns:

Reponse

Type: Object

An object that supports the following properties:

  • totalRecordCount (number)
  • moreRecords (boolean)
  • pagingCookie (string)
  • entityName (string)
  • entities (array ob BusinessEntity objects)

Example

JavaScript


RD.CRMFetchKit.FetchMore(fetchxml).then(function(response){
    console.log(response.totalRecordCount);
    console.log(response.moreRecords);
    console.log(response.entityName);
    console.log(response.pagingCookie);
    for(var i = 0; i < response.entities; i++){
        console.log(response.entities[i].getValue('lastname'));
    }
});
